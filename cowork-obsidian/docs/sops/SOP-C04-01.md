## 1 🎯 WHAT THIS LESSON GIVES YOU

By the end of this lesson, your Obsidian vault is wired to Claude Cowork — either an existing vault you already use (without reorganizing it) or a fresh vault we build together with the canonical `raw/wiki/output` shape.

Either way, you finish Lesson 1 with:

- An Obsidian vault on a local drive (NEVER inside iCloud / OneDrive / Dropbox / Google Drive)
- That vault registered in `projects/second-brain/vaults.md` with the right `mode` (`external` / `overlay` / `scaffold`)
- A safe-zones carve-out giving the `/second-brain` skill scoped read/write access to your vault — and nothing else
- For scaffold mode: a top-level `about-me/` folder with three short identity files, plus 2–3 seed raw notes for Lesson 2 to chew on
- For external/overlay mode: just the connection — your existing vault content stays exactly as it was

You install Obsidian manually (it's an app — Claude can't install it for you). Everything after the install — auto-detecting vaults, scaffolding folders, writing about-me files — Claude does from a single prompt while interviewing you.

## 2 🧠 THE BIG IDEA (READ THIS FIRST — IT'S SHORT)

A vault is just a folder of plain text files on your disk. Obsidian is a nice editor sitting on top of it. Claude Cowork is another thing sitting on top of the same folder. If either tool disappeared tomorrow, the folder — and everything in it — would still be yours.

Lesson 1 has two paths. Same destination, different starting points:

- **Path A — Connect existing vault.** You've used Obsidian before; you have a vault on this machine. We auto-detect it and connect Cowork to it. We do NOT reorganize your folders. ~5–10 minutes.
- **Path B — Create new vault.** You don't have Obsidian (or you want a fresh dedicated vault for the second brain). We install Obsidian, build the empty shell, write your about-me files, seed raw notes. ~25–40 minutes.

You don't pick. The plugin's Phase 0.5 auto-detects which path applies and asks you to confirm. If you have multiple vaults, you pick which one to connect to.

Three rules that the rest of this project depends on:

1. **Vault on a local drive.** NOT in OneDrive, iCloud Drive, Dropbox, Google Drive auto-sync, or `~/Documents` on a fresh Windows 11 install (OneDrive-synced by default). Cloud-sync corrupts vault writes — duplicate files, phantom disappearing content. Local drive only. (If you want sync across machines, that's Lesson 3 — and it uses application-layer Obsidian Sync, peer-to-peer Syncthing, or version-control Git, NOT cloud-storage folder sync.)
2. **For scaffold mode: exactly three subfolders inside every life area: `raw/`, `wiki/`, `output/`.** Same names, same spelling, same lowercase. That consistency is what lets the three Lesson 2 prompts work for every life area forever.
3. **For external mode: we do NOT touch your existing structure.** Whatever you have — PARA, Zettelkasten, daily notes — stays. The `/second-brain` skill operates on what you tell it explicitly, not by silently creating folders.

## 4 🤖 STEP 2 — RUN THE GUIDED INSTALL (RECOMMENDED)

Open Claude Cowork. Start a New task. Run the slash command:

```
/onboard-second-brain
```

That kicks off the 8-phase wizard. Phases 0 and 0.5 always run; the rest depend on which path you're on.

What happens:

- **Phase 0 — Welcome (1 min).** Soft prereq check (Project 01 is optional in v0.2.0; cowork-ai-os is optional). Sets up `projects/second-brain/`. Confirms your OS.
- **Phase 0.5 — Connect or Create (5–10 min).** Auto-detects existing Obsidian vaults from `obsidian.json`:
  - **Mac:** `~/Library/Application Support/obsidian/obsidian.json`
  - **Windows:** `%APPDATA%\obsidian\obsidian.json`
  - **Linux:** `~/.config/obsidian/obsidian.json`

  Lists every vault you've ever opened in Obsidian. The currently-open one is flagged. You pick:

  - **Pick one** → connect (mode `external`). Optional: also overlay `raw/wiki/output` on one sub-folder of it (mode `overlay`). Skip Phases 1/2/3 entirely.
  - **None of these / paste a path** → manual fallback.
  - **Create new** → fall through to Phases 1/2/3.

- **Phase 1 — Plan vaults (10 min, only if creating new).** Interview: 1 vault or several? Which 1–4 life areas? Push-back if more than 4. Saves to `projects/second-brain/vaults.md`.
- **Phase 2 — Install Obsidian (5 min, only if creating new).** Auto-detects existing app install or walks you through download.
- **Phase 3 — Scaffold first vault (15 min, only if creating new).** Picks safe local path (forbids cloud-sync). Creates life-area folders × `raw/wiki/output`. Copies `about-me/` from Cowork workspace OR interviews you for fresh identity files. Seeds 2–3 raw notes.
- **Phase 4 — Wire Cowork to vault (5 min, always runs).** Adds the vault path to `safe-zones.md` as a `/second-brain`-scoped carve-out. Tests by reading a file from the vault. **No MCP, no API key, no Local REST API plugin.** Just a permissions config update.

You'll see ~6–10 Allow prompts along the way (one per file or batch of writes). Approve each one.

> **If you've used the v0.1.0 plugin before:** Phase 0 detects your old `_aibos/state-second-brain.md` file, infers `vault_mode = scaffold` (the v0.1.0 default), and prompts once for confirmation. No data loss; pick up where you left off.

## 5 🔍 WHAT EACH MODE LOOKS LIKE WHEN YOU'RE DONE

### `external` mode (you connected to an existing vault)

`vaults.md` ends up with an entry like:

```markdown
### my-existing-vault
- name: my-existing-vault
- path: D:/dev/MyKnowledgeBase
- mode: external
- purpose: My existing PARA-organized vault
- life_areas: []   # empty for external — we don't impose structure
```

On disk: nothing changes. The plugin doesn't create any folders inside your vault. The `/second-brain` skill in Lesson 2 will ask you which folder to read raw input from and where to write wiki output — every operation is explicit.

### `overlay` mode (existing vault + raw/wiki/output on one sub-folder)

Same vault entry, but with two extra fields:

```markdown
- mode: overlay
- overlay_root: D:/dev/MyKnowledgeBase/coaching
- life_areas: [coaching]
```

On disk: the plugin created exactly three folders — `coaching/raw/`, `coaching/wiki/`, `coaching/output/` — inside the sub-folder you picked. The rest of your vault is untouched.

### `scaffold` mode (built from scratch)

The full v0.1.0 layout:

```
<vault-root>/
├── about-me/
│   ├── About Me.md
│   ├── Brand Voice.md
│   └── AI Working Style.md
├── <life-area-1>/
│   ├── raw/        ← seeded with 2–3 raw notes
│   ├── wiki/       ← empty until Lesson 2
│   └── output/     ← empty until you save deliverables
├── <life-area-2>/
│   ├── raw/
│   ├── wiki/
│   └── output/
... up to four life areas
```

## 6 ✅ MAKE SURE IT WORKED (3 MINUTES)

Three checks. Run all three regardless of which mode you ended up in.

### Check 1 — `vaults.md` exists with the right contents

Open `projects/second-brain/vaults.md`. Confirm:

- At least one vault is listed.
- Each vault has a `mode` field set to `external`, `overlay`, or `scaffold`.
- `default_vault` is set.
- For `overlay` mode: `overlay_root` points at a real sub-folder.
- For `scaffold` mode: `life_areas` is a non-empty list (1–4 entries, lowercase, hyphens).

### Check 2 — Cowork can read your vault

In your Cowork session, ask:

> *"Read one file from my vault and show me its contents."*

Cowork should:
- For `scaffold` / `overlay` mode: open one of the seeded raw notes you wrote.
- For `external` mode: ask you which file to read, then read whatever you name.

If Cowork says it can't read the path → check Phase 4 ran. The vault path needs an entry in `safe-zones.md`.

### Check 3 — Obsidian shows the same thing on disk

Open Obsidian. The file tree on the left should match what's on disk exactly.

- For `scaffold` mode: `about-me/` at the top, then your life-area folders, each with `raw/wiki/output/` inside.
- For `external` / `overlay` mode: your existing structure, unchanged. Plus, in overlay mode, the new `raw/wiki/output/` folders inside the one sub-folder you chose.

If Obsidian shows something different from disk: Obsidian is pointed at the wrong vault. Switch via **File → Open another vault → Open folder as vault → pick the path**.

## 10 🛡 SAFETY NOTE

Three warnings. Each one costs real data if ignored.

1. **Vault on a local drive, always.** Re-read Section 2's warning. OneDrive, iCloud Drive, Dropbox, and Google Drive auto-sync will eat your files if you let them. If you end up with mystery duplicates (`note.md` and `note (1).md`) or disappearing notes, sync is the cause 95% of the time. (If you want the vault on multiple machines, Lesson 3 covers the safe ways: Obsidian Sync, Syncthing, Git/GitHub.)

2. **External mode means we never reorganize your vault.** If you connected to an existing vault, the `/second-brain` skill in Lesson 2 will ask you per-write where to put output. Don't approve writes you don't want — your existing structure is your call to keep.

3. **Don't run `/onboard-second-brain` twice on the same vault expecting different behavior.** The plugin reads state and resumes. If you want to re-do a phase, edit `_aibos/state-second-brain.md` first (or ask Claude to walk you through it).

## 11 🔍 UNDER THE HOOD — FOR THE CURIOUS (TOTALLY OPTIONAL)

Skip this unless you want to know what's actually happening.

### How auto-detection works

Phase 0.5 reads Obsidian's own config file:

- **Mac:** `~/Library/Application Support/obsidian/obsidian.json`
- **Windows:** `%APPDATA%\obsidian\obsidian.json`
- **Linux:** `~/.config/obsidian/obsidian.json`

Inside, there's a `vaults` object keyed by random IDs:

```json
{
  "vaults": {
    "<random-id>": { "path": "<absolute path>", "ts": <unix-ms>, "open": true }
  }
}
```

The plugin parses this, validates each path still exists on disk, and lists them. The currently-open vault is flagged. No magic — just plain JSON.

### Why `external` mode exists

Many Obsidian users have invested years in their own structure: PARA, Zettelkasten, daily notes, MOC-driven. Forcing the v0.1.0 `raw/wiki/output × life-areas` shape onto an existing vault would destroy that work. So we added `external` mode — connect, don't reorganize, work with what's there.

The `/second-brain` skill in `external` mode asks you per-write where to put things. It's slower than `scaffold` mode by design.

### Why `overlay` mode exists

A middle path. You have an existing vault. You like its structure for most things. But you want one *sub-folder* to use the `raw/wiki/output` workflow — typically for a focused topic like coaching, a specific client, or a recurring research stream. Overlay mode adds those three folders inside one sub-folder, leaving the rest of your vault untouched.

### What the hidden `.obsidian/` folder is

Inside your vault, Obsidian creates a `.obsidian/` subfolder that holds the app's own settings (theme, plugins, view preferences). Leave it alone. The plugin's hard rules forbid the `/second-brain` skill from touching `.obsidian/`. If you back up the vault, include `.obsidian/` to preserve your settings, but otherwise ignore it.

### Why "no MCP" for the main path

For Cowork, MCP is unnecessary. Cowork has filesystem access built in — pointing it at your vault folder via `safe-zones.md` is the entire wiring. MCP is a way for *Claude Desktop* (the standalone chat app, not Cowork) to read external files. If you also want Desktop access, that's Lesson 3's optional challenge — and it does require an MCP server.

## 12 🎓 TEACHING NOTES (FOR THE INSTRUCTOR)

**Timing:** 5–15 min for connect-existing students; 25–40 min for scaffold-fresh students. Plan for both in the cohort session.

**Live demo:** Open the auto-detect output on a populated machine before students run it. The "wait, it just found my vault?" moment is the hook.

**Emphasis:** *"Local drive. One vault per context. If you have an existing vault, we connect — we don't reorganize."* That's the structural rule set.

**Watch for:**

- Students on Windows with OneDrive enabled. Check their vault path before they proceed. `Documents` is OneDrive-synced by default on fresh Windows 11 installs.
- Students who already have Obsidian and a real vault but jump to the "create new" branch out of habit. **Push them to connect-existing.** External mode preserves their work.
- Students who want to use their existing note-taking app's database as the vault. Push back — Obsidian needs a folder of `.md` files. Convert the export, don't try to live-mount the foreign app.
- Students who skip the seed notes in scaffold mode. Lesson 2 will be awkward without material. Make them do it.
- Mac students who hit the Files-and-Folders permission prompt and click Deny by accident. Walk them through System Settings → Privacy & Security → Files and Folders to fix.
